# LocalBook — 需求文档

## 功能描述

一个运行在浏览器中的本地密码管理器，用户注册登录后，可增删查改各平台账号密码。所有密码使用对称加密后存储在浏览器 localStorage 中，可导出为 JSON 文件备份或迁移。附带备忘录（日历 + 农历）、便签和联系人管理功能。

## 技术路线

- 前端：原生 HTML + CSS + JavaScript（零依赖，双击 `index.html` 即可运行，或通过 web 服务器在手机端访问）
- 数据存储：localStorage，JSON 格式序列化，键名 `pbook_data`
- 加密算法：
  - 登录凭证：SHA-256 哈希（不可逆）
  - 存储密码：AES-256-GCM 对称加密，密钥经 PBKDF2（10 万次迭代）派生
- 数据迁移：支持导出 / 导入 `pbook_data.json` 文件

## 详细需求

### 登录 / 注册

- 登录页面：输入用户名、密码，验证后进入管理页
- 注册页面：输入用户名、登录密码、加密令牌（Token）、确认令牌
- 注册后用户名和登录密码经 SHA-256 哈希后存入 localStorage
- 加密令牌同样经 SHA-256 哈希存储，用于后续验证
- 页面间可切换（登录 ⇄ 注册）
- 表单校验：用户名 ≥ 2 字符，密码 / 令牌 ≥ 4 字符
- 同名用户不可重复注册
- 记住密码：登录页提供复选框，勾选后将用户名密码 Base64 编码存入 localStorage，下次自动填充
  - 取消勾选后登录则清除已保存的凭据

### 密码管理页面

- 顶栏：左侧显示 logo + 用户名 + 版本号（`v1.2.0`），右侧实时时钟 + 退出登录按钮
- 操作按钮：新增密码、编辑密码、导出数据、退出登录
- 密码列表表格：名称、地址、用户名、操作列
- 操作列：「解密」按钮弹窗解密、「删除」按钮确认后删除
- 空状态：无密码时显示提示文字

### 新增 / 编辑密码

- 模态框表单：名称、地址、用户名、密码、加密令牌
- 保存前输入加密令牌，验证通过后 AES 加密存储
- 编辑时地址和用户名锁定不可修改
- 令牌验证失败则拒绝保存

### 解密密码

- 模态框展示该条记录的地址和用户名
- 输入加密令牌，验证通过后 AES 解密展示真实密码
- 令牌错误提示「Token 错误」
- 可点击「隐藏」关闭明文展示
- 支持 Enter 快捷提交

### 数据导入 / 导出

- 导出：下载 `pbook_data.json`，包含完整数据（users + passwords + memos + contacts + stickies）
- 导入：选择本地 JSON 文件，校验格式后合并至当前数据
  - 用户名冲突时保留缓存用户，跳过导入中的同名用户
  - 仅新增用户及其密码、联系人
  - 备忘录已有日期不覆盖
- 导入数据仅限登录页操作，管理页不提供导入入口

### 会话保持

- 使用 sessionStorage 缓存登录状态
- 页面刷新后自动恢复登录（不丢失当前会话）
- 退出登录清除会话

### 备忘录

- Tab 栏支持切换「备忘录」「便签」「联系人」「密码管理」
- 日历视图：月历表格，头部支持上/下月导航
- 每个日期格子显示：
  - 公历日期数字
  - 农历月日（如「四月初十」）
- 当日日期高亮（圆形背景）
- 有备忘录的日期显示小圆点标记
- 点击日期弹出编辑模态框
- 编辑框标题显示完整公历 + 农历（如「2026年5月26日 星期二 农历四月初十」）
- 支持保存 / 取消 / 遮罩关闭

### 农历转换

- 内置农历数据（1900–2100 年），离线计算
- 编码：bits 0–3 闰月，bits 4–15 12 个月大小（1=30 天），bit 16 闰月大小
- 核心函数：
  - `lYearDays(y)` — 农历年总天数
  - `leapMonth(y)` — 闰月月份（0=无）
  - `leapDays(y)` — 闰月天数
  - `solarToLunar(y,m,d)` — 公历转农历，返回 `{ monthStr, dayStr }`

### 联系人管理

- Tab 栏第三个标签「联系人」
- 新增联系人按钮位于联系人视图顶部
- 联系人以卡片网格展示，自适应列数
- 卡片包含：姓名（加粗）、昵称（灰色小字）、性别标签、手机/座机/邮箱/地址（带图标行）、年龄、备注
- 卡片底部：编辑、删除按钮
- 编辑弹窗表单：姓名（必填）、昵称、手机、座机、邮箱、年龄、性别（下拉）、地址、备注
- 联系人数据以明文存储，不加密

### 便签

- Tab 栏第二个标签「便签」，位于「备忘录」与「联系人」之间
- 新增便签按钮位于顶栏操作区（与「新增密码」同排），tab 切换时显隐
- 便签以彩色卡片网格展示，自适应列数
- 每张卡片展示：标题（可选）、正文内容、创建/更新时间、编辑/删除按钮
- 卡片背景色支持 7 种预设颜色（黄、绿、蓝、粉、紫、橙、青）
- 编辑弹窗表单：标题（可选）、内容（必填）、颜色选择器（单选圆点）
- 便签数据以明文存储，不加密
- 导出/导入时便签随用户数据一并迁移

### 界面要求

- 渐变背景登录页（紫色渐变）
- 圆角卡片式布局，柔和阴影
- 响应式设计，适配手机端（640px 断点）
- Toast 轻提示反馈操作结果
- 模态框动画（淡入 + 上滑）
- 遮罩点击可关闭模态框
- 实时时钟：顶栏右上角显示当前日期时间，每秒更新

## 部署与本地访问

### 电脑端

直接双击 `index.html` 在浏览器打开即可，无需任何服务器。

### 手机端（同一 Wi-Fi）

由于 localStorage 在 `file://` 协议下部分浏览器不支持，需在项目目录启动一个静态文件服务器。

**1) Node.js（推荐）**

```bash
# 如未安装 serve，先全局安装
npm i -g serve
serve .

# 或使用 npx（部分系统需先装 serve 到项目）
npx serve .
```

**2) Python**

```bash
# Python 3
python -m http.server 8080

# Python 2
python -m SimpleHTTPServer 8080
```

启动后，手机和电脑连接同一 Wi-Fi，手机浏览器访问 `http://电脑IP:端口` 即可（电脑 IP 可用 `ipconfig` 或 `ifconfig` 查看）。

### 服务器长期部署

项目为纯静态文件，可部署到任何 Web 服务器。

**使用 pm2 + serve 长期运行**

```bash
npm i -g serve pm2
pm2 start serve --name localbook -- . -p 8080
pm2 save
pm2 startup
```

**静态托管平台（推荐）**

| 平台 | 部署方式 |
|---|---|
| Vercel | `npx vercel --prod` |
| Netlify | 拖拽项目文件夹到 Netlify Drop |
| GitHub Pages | 推送到 `gh-pages` 分支 |

部署后获得公网 URL，手机直接访问该 URL 即可使用。

### PWA 安装

应用已支持 PWA（Progressive Web App），安装后可全屏独立运行，离线也可打开：

- **Android Chrome**：打开网站 → 弹出「添加到主屏幕」提示 → 确认
- **iPhone/iPad Safari**：打开网站 → 底部分享按钮（📤）→「添加到主屏幕」→ 添加

从桌面图标打开后无浏览器地址栏，体验接近原生 App。

## 使用流程

1. 双击 `index.html` 或通过 web 服务器访问
2. 注册账号（设置用户名 + 登录密码 + 加密令牌）
3. 登录进入管理页
4. 点击「新增密码」→ 输入地址 / 用户名 / 密码 → 输入加密令牌 → 保存
5. 点击「解密」→ 输入加密令牌 → 查看明文密码
6. 切到「备忘录」tab → 点击日期 → 记录每日事项
7. 切到「便签」tab → 新增彩色便签，自由记录灵感
8. 切到「联系人」tab → 新增或编辑联系人
9. 定期点击「导出」备份数据；迁移时在新设备「导入」
